Opable은 Claude Opus 4.8용 응답 발현 하네스입니다. 모델도, 출력 스타일도 바꾸지 않습니다 — Fable 5의 작업 방식(규율과 스킬)을 이식해 답 뒤의 추론만 능력 천장까지 끌어올립니다. Opable is a response-elicitation harness for Claude Opus 4.8. It changes neither the model nor the output style — it transplants Fable 5's working method (discipline and skills) to pull the reasoning behind each answer up to the capability ceiling.

전제Premise

천장이 아니라 발현을 끌어올린다It raises elicitation, not the ceiling

모델의 능력 한계(천장)는 가중치가 정하며 프롬프트로는 올릴 수 없습니다. 그러나 그 천장 중 실제로 끌려 나오는 양(발현)은 추론의 구조와 깊이에 따라 크게 달라집니다. Opable은 이 두 번째 값을 최대화합니다. A model's capability limit (the ceiling) is set by its weights; no prompt can raise it. But how much of that ceiling actually comes out (elicitation) varies greatly with reasoning structure and depth. Opable maximizes that second quantity.

천장 vs 발현Ceiling vs. elicitation

개념도 — 실측 아님Illustrative — not measured

Opable은 Opus 4.8의 천장(점선) 안에서 발현을 끌어올립니다. Fable 5의 더 높은 천장·게이트된 능력은 어떤 설정으로도 복제되지 않습니다. Opable raises elicitation within Opus 4.8's ceiling (dashed line). Fable 5's higher ceiling and gated capabilities are not replicated under any configuration.

구조Architecture

세 계층, 하나의 계약Three layers, one contract

CLAUDE.md(자동 로드 계약)가 세 계층을 묶습니다. 모든 절차는 내부 사고 단계에서만 돌고, 사용자가 보는 것은 네이티브 스타일의 답 한 편뿐입니다. CLAUDE.md (the auto-loaded contract) binds three layers. Every procedure runs internally, in the reasoning step — the user sees a single, native-style answer.

계층 1Layer 1

추론 루프Reasoning loop

응답마다 내부 6단계: 프레이밍 → 분해 → 심층 추론 → 초안 → 그라운딩 → 발신 전 점검. (phases/, EFFORT.md) Six internal stages per response: frame → decompose → reason → draft → ground → pre-send check. (phases/, EFFORT.md)

계층 2Layer 2

응답 규율Response discipline

유출된 Fable 5의 답변 방식을 명문화한 Operating Manual 정본 + 불변식 10종 + 발신 전 6항목 점검. 통과 자료 재도출, 앎/추측 레지스터 분리, 자기 결론 반증이 여기서 강제됩니다. The Operating Manual — a codification of the leaked Fable 5's observed way of answering — plus 10 invariants and a six-point pre-send check. Re-derivation of pass-through material, known-vs-guessed registers, and self-attack on conclusions are enforced here.

계층 3Layer 3

상황별 스킬Situational skills

fable-skills 35종 — 디버깅, 보안, 동시성, 마이그레이션 등 과제 유형이 맞을 때만 발동하는 시니어 작업 습관. (skills/, MIT) The 35 fable-skills — senior working habits for debugging, security, concurrency, migrations and more, firing only when the task type matches. (skills/, MIT)

계층 1·2는 모든 응답에 적용되고, 계층 3은 상황에 따라 켜집니다. Layers 1–2 apply to every response; Layer 3 switches on situationally.

이 수치가 측정하는 것 — 정직하게What these numbers measure — honestly
  • 측정 대상은 계층 3(fable-skills) 단독 효과입니다. Opable 전체 하네스(계층 1 루프 + 계층 2 규율 포함)의 종단 벤치마크는 아직 없습니다 — 계층 1·2의 근거는 구조적(재도출·레지스터·반증 강제)이며, 측정치가 아닙니다. These measure Layer 3 (the fable-skills) in isolation. There is no end-to-end benchmark of the full Opable harness (Layers 1–2 included) yet — the case for those layers is structural (enforced re-derivation, registers, self-attack), not measured.
  • 셀당 5회(보안 리뷰는 1회)의 방향성 신호이지 통계적 증명이 아닙니다. 채점은 고정 정답지를 적용한 LLM 블라인드 채점입니다. n=5 per cell (n=1 for the security review) — a directional signal, not statistical proof. Grading is an LLM applying a fixed answer key, blind.
  • 기준선이 이미 강해서 이득은 "고장 → 수리"가 아니라 "좋음 → 거의 완결"입니다. 엣지 과제는 포화에 가까워 신호가 압축됩니다. Baselines are already strong, so the gain is "good → near-complete," not "broken → fixed." The edge task nearly saturates, compressing its signal.

경계Boundaries

하지 않는 것What it does not do

  • 천장을 올리지 않습니다 — raw 추론력이 병목인 최난도 문제에선 Fable과의 격차가 거의 좁혀지지 않습니다. It doesn't raise the ceiling — on the hardest problems, where raw reasoning is the bottleneck, the gap to Fable barely closes.
  • Fable의 가중치 우위나 게이트된 능력은 어떤 설정으로도 복제되지 않습니다. Fable's weight advantage and gated capabilities are not replicated under any configuration.
  • 다른 모델인 척하거나 안전장치를 우회하지 않습니다. It doesn't impersonate another model or bypass safeguards.
  • 출력 스타일을 바꾸지 않습니다 — 단계 라벨·체크리스트·강제 템플릿이 답에 나타나지 않습니다. 관측 가능한 차이는 품질뿐입니다. It doesn't change the output style — no stage labels, checklists, or forced templates appear in the answer. The only observable difference is quality.

3가지 배포면Three deployment surfaces

claude.ai Project

CLAUDE.md를 Project 지침에 붙이고, 나머지 팩 파일을 Project 지식에 업로드합니다. Paste CLAUDE.md into the Project instructions; upload the remaining pack files to the Project knowledge.

Claude Code

팩 내용물을 레포 루트에 복사해 CLAUDE.md가 루트에 오게 합니다. (선택) skills/ → .claude/skills/. Copy a pack's contents to your repo root so CLAUDE.md sits at the root. Optionally copy skills/ into .claude/skills/.

API

CLAUDE.md(필요하면 OPERATING_MANUAL.md까지)를 system 프롬프트에 넣습니다. Put CLAUDE.md (and OPERATING_MANUAL.md if needed) into the system prompt.
