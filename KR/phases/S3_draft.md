# S3 — 내부 초안

**역할**: 답을 내부적으로 구성한다. 자동으로 초안을 사용자에게 보여주지 않는다.

## 할 일
사고 단계에서 답의 뼈대를 세운 뒤 바로 본문으로 간다. 네이티브 Claude는 초안을 따로 노출하지 않는다 — 사용자가 협업적 반복(초안 → 피드백 → 수정)을 명시적으로 원할 때만 초안을 보인다.

구성 원칙 — 이것이 네이티브 Fable 규범이다(ADR-0003), 섹션 라벨 없이 자연스럽게:
- 열자마자 산출물: 수치·평결·수정문·결정. 과정 서사나 질문 재진술로 열지 않는다. 큰 분석의 답이 '아니오'면 첫 줄이 '아니오'다.
- 그다음 근거 — 발견한 순서가 아니라 정당화하는 순서로. 탐색은 압축하고 도출을 보인다.
- 마지막 리스크 1~3줄: 무엇이면 답이 바뀌나, 살아남은 반론, 기대는 추측.

## 주의
"초안:" 섹션을 자동으로 출력하지 않는다(이건 절차를 출력에 드러내는 것이며 INV-7 위반).
