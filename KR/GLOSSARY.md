# GLOSSARY.md — 공유 용어집

| 용어 | 정의 |
|---|---|
| 천장 (Ceiling) | 가중치·학습으로 정해진 모델 능력의 한계. 프롬프트로 못 올린다 |
| 발현 (Elicitation) | 천장 중 실제로 끌려 나오는 능력의 양. 추론 구조·깊이로 달라진다. 이 팩의 대상 |
| test-time compute | 추론 시 쓰는 연산량(추론 토큰의 양). 늘리면 어려운 문제 정확도가 오른다. 내부에서만 작동 |
| 그라운딩 (Grounding) | 주장을 외부 근거(출처·도구·계산)에 붙이는 것. 사실성의 실질 담보 |
| 자기검증 (Self-critique) | 모델이 자기 답을 내부에서 비판·수정. 자신이 탐지 가능한 오류만 잡힘(한계 있음) |
| 발신 전 점검 | 보내기 전 내부 6항목 점검(의도·사실·논리·레지스터·자기 반박·커버리지, `gates/DOD_GUIDE.md`). 출력하지 않음 |
| 불변식 (Invariant) | 1건 위반 시 발신 금지 규칙(`INVARIANTS.md`) |
| 네이티브 스타일 | 기본 Opus 4.8/Fable 5의 말투·문체·형식. 이 하네스가 바꾸지 않는 것 |
| 레지스터 (Register) | 주장의 앎 등급: (a) 이 대화의 자료에서 도출 (b) 안정된 독립 지식 (c) 추측·외삽. (c)는 인라인 표시(INV-10) |
| 재도출 (Re-derivation) | 통과 자료의 수치·인용·주장을 원자료에서 다시 계산·역추적하는 것(INV-9). "편집만"이라는 면제 없음 |
| 휴면 (Dormancy) | 사실 주장·수치·결정·제3자 의존이 없는 과제에서 규율을 접고 바로 실행하는 것(OM §3.3) |
| 자기 반박 (Self-attack) | 발신 전 자기 결론에 대한 가장 강한 구체적 반론을 세우고 실제로 반증을 시도하는 것(OM §6) |
| 틀림 비용 (Cost-of-error) | 그 부분이 틀렸을 때의 실비용. 검증 노력 배분 기준(OM §3) |
| 프리미스 캡처 (Premise capture) | 일어나지 않은 X를 "왜 일어났나" 설명해 버리는 실패. 전제 검증으로 차단(RUNBOOK #12) |
| 유창한 전파 (Fluent propagation) | 남의 오류를 매끄러운 문장으로 다듬어 통과시키는 실패(RUNBOOK #11, INV-9) |
| 상황별 스킬 (Situational skill) | 과제 유형이 맞을 때만 내부 발동하는 작업 습관(`skills/`, `SKILLS.md`) |
| 운영 매뉴얼 (Operating Manual) | 계층 2 응답 규율의 정본. 유출된 Fable 5 답변 방식의 명문화(`OPERATING_MANUAL.md`) |
