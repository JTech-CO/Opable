# ADR-0004: fable-skills 35종을 계층 3(상황별 스킬)으로 병합한다

- **상태**: 채택
- **일시**: 2026-07-08
- **관련**: skills/, SKILLS.md, 내부 단계 S0·S1, DOD_GUIDE, ADR-0003

## 맥락
fable-skills(MIT, adamentwistle/fable-skills)는 Fable이 경량 모델을 자기 작업 수준으로 끌어올리기 위해 쓴 35종의 엔지니어링 규율 스킬이다. 벤치마크에서 스킬 적용 시 Opus 4.8이 전 영역에서 상승했고(디버깅 5.2→7.8/8 등) 분산도 줄었다. 기존 하네스는 보편 규율만 있고 과제 유형별 절차가 없었다.

## 결정
35종 전량을 루트 `skills/`에 벤더링한다(영문 원문 그대로, 출처·MIT 표기). 각 팩에 `SKILLS.md` 색인(한/영)을 두어 상황→스킬 매칭, 적용면(대화+코딩/코딩 중심/에이전트 세션) 분류, 기존 불변식·OM과의 대응을 기록한다. 발동은 S0/S1에서 과제 유형을 스킬 description과 대조해 해당분만 로드한다. Claude Code에서는 `skills/`를 `.claude/skills/`로 복사하면 Skill 도구로도 발동할 수 있다.

## 근거
스킬은 "항상 로드"가 아니라 상황 발동이라, 상시 컨텍스트를 늘리지 않으면서 유형별 발현을 키운다. 전량 유지는 벤치마크가 세트로 검증되었고, 적용면 분류로 배포별 취사선택이 가능하기 때문이다.

## 결과 / 트레이드오프
- 이득: 상시 컨텍스트 비대화 없이 과제 유형별 발현이 커진다.
- 비용: 스킬 본문이 영문뿐이다(모델이 읽는 파일이므로 실용상 문제 없음, 유지비 절반). 업스트림과의 표류 가능성이 있다(벤더링 시점 고정, 출처 링크로 추적).

## 검토한 대안
- 35종 전부 KR 번역 — 유지비 2배, 수혜자는 모델뿐. 기각.
- 일부만 선별 — 세트 검증이 훼손되고, 적용면 분류로 충분. 기각.
- phases에 인라인 통합 — 상시 컨텍스트가 비대해지고, 상황 발동이라는 설계 자체를 부정. 기각.
