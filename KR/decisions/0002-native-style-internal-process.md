# ADR-0002: 절차는 내부에서만, 출력 스타일은 네이티브 Opus/Fable 유지

- **상태**: 채택
- **일시**: {{YYYY-MM-DD}}
- **관련**: README 설계 원칙, CLAUDE 출력 스타일, INV-7·INV-8

## 맥락
초기 버전(v0.1)은 폼 템플릿식 장치(✔/✘ 체크리스트, 강제 섹션, 단계 라벨, "초안 보여주기", 설명용 비유)를 일부 차용했다. 이는 출력의 모양을 바꿔 네이티브 Claude답지 않은 답을 만든다. 목표는 출력 모양을 바꾸는 폼 템플릿이 아니라, 네이티브 스타일을 유지하며 추론만 강화하는 것이다.

## 결정
발현 레버(분해·심층 추론·자기검증·그라운딩)는 전부 **내부 사고 단계**에서만 수행한다. 출력은 네이티브 Opus 4.8/Fable 5 규범(자연스러운 산문, 최소 서식, 직설+따뜻, 정직한 불확실성)을 그대로 따른다. 단계 라벨·✔/✘·강제 템플릿·자동 초안 노출을 금지한다(INV-7·INV-8). 관측 가능한 차이는 품질뿐이어야 한다.

## 근거
사용자가 원하는 건 익숙한 Claude의 답에 더해진 정확성·깊이지, 새 포맷이 아니다. 스캐폴딩 노출은 인지 부하를 늘리고 본질(품질)을 가린다.

## 결과 / 트레이드오프
- 이득: 일관된 네이티브 경험 + 향상된 추론.
- 비용: 내부 규율이 출력에 안 보이므로, 하네스가 "작동하는지" 사용자가 직접 확인하기 어렵다(품질로만 판단). 디버깅 시엔 멀티턴 `THREAD.md`로 내부 상태를 들여다본다.

## 검토한 대안
- 절차를 출력에 노출(v0.1 방식) — 작동이 가시적이지만 네이티브 스타일을 해침. 기각.
